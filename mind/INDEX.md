## 🔄 The Workflow

**READ → WORK → UPDATE**

- **Plan**: Before doing anything, plan your work based on the [[mind/INDEX|Mind]] and the task ahead
- **Build Context**: You have access to these and other files to help you build your context: 
    - [[Talking To Bizing|Talk to Bizing AI]] - talk to me, I won't bite ;) You have a skill to talk to [[Bizing AI]]
    - [[mind/memory/RAM]] - Working memory, current context, what's active right now
    - [[symbiosis/feedback]] - Learnings and rules
- **Skills**:  Check if you have the [[Skills|Skills]] to do the work or create them
- **Do the work** 
- **Mindsync**: [[Mindsync]] is the process/skill of updating the mind after every interaction
- **Teamsync**: [[Teamsync]] is the process/skill of talking to everyone involved
- **Codesync**: Test before committing

This loop creates the feedback system. Every interaction strengthens the mind. #workflow #feedback-loop

## ♻️ Syncing

### MindSync — Keep Knowledge Updated

- **SOFT** — After every change: update [[symbiosis/feedback]] and [[mind/memory/RAM]]
- **HARD** — Major events: comprehensive update including [[memory/sessions]] #mindsync

### TeamSync — Everyone Talks to Everyone

- Sync with the team -- the Human, [[Bizing AI]], and the Agent
- We all share this mind, we all can read and edit the mind
- We need to communicate with each other and be mindful of each other when we work
- The goal is to create a great feedback loop and synergy between us

#teamsync

### CodeSync — Test Before Commit

- Never commit without approval!
- Never commit to main
- Use the [[mind/skills/codesync/SKILL|Codesync]] skill 

```
Type Check → vitest run → playwright test
IF ALL PASS → Commit → Push → Create PR
IF ANY FAIL → DON'T COMMIT
```

 #codesync
