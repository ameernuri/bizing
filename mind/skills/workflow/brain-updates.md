# Brain Updates

*When and how to update the brain.*

---

## When to Update

**Always document:**
- ✅ Significant features built
- ✅ Architecture decisions made
- ✅ New patterns established
- ✅ Important fixes with learnings
- ✅ Business model changes
- ✅ Agent capabilities added

**Don't document:**
- ❌ Minor bug fixes
- ❌ Dependency updates (unless major)
- ❌ Typo fixes
- ❌ Formatting changes

---

## What to Update

### Standup (`symbiosis/standup.md`)
- Current work
- Blockers
- Next steps

### Session Logs (`memory/sessions/`)
- Detailed work log
- Files changed
- Decisions made

### Knowledge (`knowledge/`)
- Domain understanding
- Technical patterns
- API documentation

### Decisions (`symbiosis/decisions`)
- Significant choices
- With reasoning
- With code links

---

## Quality Over Quantity

The brain should reflect **what matters** —
the evolution of Bizing as an entity,
not the noise of every code change.
