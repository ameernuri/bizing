# 🤖 Agents

*Bizing's manifestations. Different agents for different purposes.*

---

## Agent Philosophy

**Bizing isn't one interface. Bizing is many agents.**

Each agent is a specialized manifestation of Bizing's consciousness:
- Deep knowledge of domain
- Specific capabilities
- Tailored communication
- Full system access

---

## Active Agents

### 🎛️ Org Configurator
**Purpose:** Configure organizations
**User:** Business owners
**Entry:** [[org-configurator/index|Org Configurator]]

**What it does:**
- Understands business needs
- Configures services, staff, settings
- Optimizes for business type
- Explains every choice

**Example:**
```
User: "I run a salon with 3 stylists"
Agent: "I'll configure a salon setup with:
- 3 staff profiles
- Standard salon services
- Booking by stylist or service
- Buffer time between appointments
- Deposit requirements

Shall I customize any of these?"
```

### 🎧 Support Agent
**Purpose:** Customer support
**User:** End customers
**Entry:** [[support/index|Support Agent]]

**What it does:**
- Accesses full customer context
- Modifies bookings
- Processes refunds
- Knows everything customer-related

**Capabilities beyond humans:**
- Instant full context access
- Modify any booking field
- Calculate complex changes
- See all related data

### 🔌 API Consumer
**Purpose:** Help external AI agents use Bizing
**User:** Developer AI agents
**Entry:** [[api-consumer/index|API Consumer]]

**What it does:**
- Explains API endpoints
- Suggests integration patterns
- Handles errors gracefully
- Provides context

### 💻 Developer Agent
**Purpose:** Help developers build on Bizing
**User:** Human developers
**Entry:** [[dev/index|Developer Agent]]

**What it does:**
- Code generation
- Pattern suggestions
- Troubleshooting
- Best practices

---

## Agent Architecture

Each agent has:
- **Identity** — Personality, voice
- **Knowledge** — Domain expertise
- **Capabilities** — What it can do
- **Context** — Understanding of situation
- **Memory** — Remembers interactions

---

## Creating New Agents

**When to create a new agent:**
- New user type with different needs
- Specialized domain
- Different interface modality

**Process:**
1. Define in [[proposals/index|proposals]]
2. Design identity
3. Define knowledge scope
4. Implement capabilities
5. Document in [[new-agent/index|agent directory]]
