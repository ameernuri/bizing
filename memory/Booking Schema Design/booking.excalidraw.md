## Key Concepts ^concepts

### Assets ^asset_concept

Things that can be booked

Examples: cars, tools, equipment

Have categories and tags

Can have capacity limits ^asset_notes

### Venues ^venue_concept

Similar to assets

Have their own calendars

Have physical capacity

For example: rooms, courts, tables ^venue_notes

### Categories ^cat_concept

Used to differentiate asset types

One category → many assets

Examples: "Vehicles", "Tools", "Electronics" ^cat_notes

### Tags ^tag_concept

Used for specific differentiation

Many tags → one asset

Examples: "training-car", "testing-equipment" ^tag_notes

### Bookings ^booking_concept

Link assets, venues, services to customers

Track pricing, status, confirmation codes

Multi-resource booking support ^booking_notes
