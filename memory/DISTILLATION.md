## Critical Rules

**Never commit or push without explicit confirmation.** See [daily/2026-02-09](./daily/2026-02-09.md)

**Working safely on a big project:**
- Work on a branch, never touch main directly
- Never commit without asking for confirmation
- Never push to main without approval
- Never build features unasked
- Ask for clarification when ambiguous
- Make small, surgical changes

See [WORKFLOW.md](./WORKFLOW.md)

---

## Design Approach

**Memory is for exploration, not detailed specs.** See [daily/2026-02-09](./daily/2026-02-09.md)

**Memory includes:**
- High-level concepts
- Bullet points, open questions
- Design decisions and trade-offs

**Memory excludes:**
- Detailed field definitions (code is the source of truth)
- Code duplicates

See [daily/2026-02-09](./daily/2026-02-09.md)
